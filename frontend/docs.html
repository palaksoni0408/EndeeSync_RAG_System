EndeeSync Documentation

A production-grade Retrieval-Augmented Generation (RAG) system powered by Endee vector database. EndeeSync grounds LLM responses in your own authoritative documents to eliminate hallucinations.

Core Features

Semantic Search

Uses 384-dimensional embeddings (sentence-transformers) to find meaning beyond keywords.

Endee Vector DB

Sub-millisecond HNSW indexing with INT8 quantization for efficient storage.

Smart Ingestion

Intelligent chunking for PDFs (with overlap) to preserve context across boundaries.

Robust Fallback

Three-tier LLM chain: OpenAI GPT-3.5 -> Groq -> Local Models.

System Architecture

The system consists of three main pipelines: Ingestion, Storage, and Retrieval.

┌─────────────────────────────────────────────────────────────┐ │ INGESTION PIPELINE │ └─────────────────────────────────────────────────────────────┘ │ Raw Documents ────────────┤ (txt/md/pdf) │ ▼ ┌────────────────┐ │ ingest.py │ Load & chunk documents │ chunk_size: │ with overlap │ 512 tokens │ └────────┬───────┘ │ ▼ ┌────────────────┐ │ embed.py │ Generate embeddings │ all-MiniLM- │ (384 dimensions) │ L6-v2 │ └────────┬───────┘ │ ▼ ┌────────────────┐ │ store.py │ Batch upsert to Endee │ (max 1000 │ with metadata │ per batch) │ └────────┬───────┘

Installation

EndeeSync is designed to run locally. Prerequisites include Python 3.8+ and Docker (for the Endee server).

1. Clone and Setup

git clone https://github.com/bytebender77/endee-palak.git
cd endee-palak
pip install -r requirements.txt

2. Configure Environment

Create a .env file in the root directory:

ENDEE_BASE_URL=http://localhost:8080/api/v1
OPENAI_API_KEY=sk-...

Usage

Ingesting Documents

Place your PDF or Markdown files in the data/raw_docs/ directory and run:

python app.py --ingest

Running the Web Interface

./run.sh

This will start the FastAPI server on port 8000. You can then access the dashboard at http://localhost:8000/dashboard.

API Reference

We provide a full Swagger UI for interactive API documentation. You can test endpoints for ingestion, search, and RAG generation directly from the browser.
